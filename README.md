# Tinkoff-cashback-analysis
# Анализ программы кэшбэка Тинькофф

Исследование поведения клиентов и эффективности программы кэшбэка банка Тинькофф на основе транзакционных данных. Проект включает анализ влияния коммуникаций, трат, обновления мобильного приложения и структуры покупок на получаемый кэшбэк.

## Цели проекта

1. Изучить данные о тратах, категориях покупок и начисленном кэшбэке клиентов.
2. Проверить, существует ли связь:
   - между коммуникациями и величиной кэшбэка,
   - между оборотом клиента и суммой полученного кэшбэка.
3. Оценить влияние обновления мобильного приложения на:
   - структуру трат,
   - динамику кэшбэка,
   - категории покупок.
4. Выявить, какие категории дают основной вклад в кэшбэк до и после обновления.
5. Провести статистическую проверку значимости изменений.
6. Сформулировать и проверить собственную гипотезу с использованием бутстрап-методов.

## Что было сделано

1. **Предобработка данных**
   - проверены типы и распределения признаков;
   - удалены отрицательные значения;
   - обработаны выбросы (грубые выбросы в тратах и кэшбэке).

2. **Анализ коммуникаций**
   - выполнены корреляции между коммуникациями и кэшбэком;
   - вывод: связи практически нет (коэффициенты близки к 0).

3. **Связь трат и кэшбэка**
   - рассчитана суммарная трата (`total_spend`) и общий кэшбэк (`total_cashback`);
   - выявлена слабая положительная корреляция (~0.26–0.28).

4. **Анализ категорий**
   - определены категории, наиболее связанные с кэшбэком («Electronics», «Supermarkets», «Fastfood»);
   - выполнено сравнение категорий до/после обновления.

5. **Оценка влияния обновления приложения**
   - построены распределения кэшбэка до и после обновления;
   - рассчитаны средние значения по категориям;
   - построен бутстрап 1000+ выборок;
   - проведён парный t-test по всем категориям;
   - вывод: в большинстве категорий обновление привело к статистически значимому росту кэшбэка.

6. **Проверка собственной гипотезы**
   - гипотеза: «Пользователи получают больше банковского кэшбэка, чем партнёрского»;
   - бутстрап + t-test подтвердили гипотезу (p-value ≈ 0).

## Стек

- Python  
- NumPy, Pandas

- SciPy (t-test)

- Jupyter Notebook  


## Основные выводы

- Коммуникации **не влияют** на кэшбэк.  
- Чем больше клиент тратит, тем больше получает кэшбэка (но связь слабая).  
- Обновление приложения оказало **значимый положительный эффект**.  
- Большинство категорий показали рост.  
- Клиенты получают **больше банковского кэшбэка**, чем партнёрского.
